# 🌼 희희 (Heehee) — Happy Quokka Desktop Pet

> 맥북 코너에 조용히 살고 있는 픽사 스타일 3D 쿼카.
> 숫자도 없고, 닫기 버튼도 없고, 그냥 같이 있는 존재.

| 항목 | 내용 |
|------|------|
| 이름 | 희희 (Heehee) |
| 기본 표정 | 항상 웃음 — 입꼬리 올라간 픽사 스마일, 눈은 살짝 초승달 |

## 컨셉

희희는 게임 캐릭터가 아니라 **룸메이트**. 관리 대상이 아니라 같이 사는 존재.
내가 일하면 옆에서 나를 보고, 맥북 충전하면 희희도 쉰다.
수치도, 미션도, 알림도 없다. 그냥 있다.

---

## 상태 시스템

| 상태 | 트리거 | 행동 | 비고 |
|------|--------|------|------|
| `AWAKE` | 기본 / 충전 해제 후 / 앱 시작 시(비충전) | 화면 응시, 눈 깜빡임, 귀 쫑긋, 항상 웃음 | 앱 시작 시 충전 상태 확인 → 충전 중이면 CHARGING_IDLE로 |
| `HAPPY` | 쓰다듬기 / 풀 먹은 직후 | 통통 튀기, 하트 이펙트 → 2초 후 AWAKE | |
| `IDLE_GESTURE` | AWAKE 상태에서 랜덤 (가끔) | 풀을 쳐다보며 "먹고 싶다" 제스처 → 안 줘도 AWAKE로 | 강제성 없음 |
| `CHARGING_IDLE` | 충전 연결 직후 / 앱 시작 시(충전 중) | 풀밭 배경 fade-in, 털썩 앉아 멍때리기 | |
| `NAPPING` | 충전 5분 경과 | 눈 감고 낮잠, 민들레 꼭 쥔 채로, ZZZ 이펙트 | |
| `STRETCHING` | 충전 해제 직후 | 기지개 + 눈 비비기 → AWAKE 복귀 | |
| `DROWSY` | 밤 11시 이후 (Phase 2) | 졸린 표정, 하품 | Step 6에서 구현 |

**Note:** 배고픔(HUNGRY) 게이지/타이머 없음. `IDLE_GESTURE`로 자연스럽게 표현.
**Debounce:** 충전 상태 변경은 2초 이내 재변경 무시 (MagSafe 빠른 탈착 대응).

## 인터랙션

**유저가 할 수 있는 것**
- 쓰다듬기 (클릭)
- 풀 먹이기 (버튼) — 강제성 없음, 안 줘도 슬퍼지지 않음
- 이름 짓기 (최초 1회, 기본값: 희희, 트레이 메뉴 + 창 title에 표시)
- 창 위치 이동 (드래그)

**희희가 알아서 하는 것**
- 충전 감지 → 자동 휴식 모드 전환
- 충전 해제 → 기지개 후 AWAKE 복귀
- 시간대 반응 (밤 11시 이후 졸린 표정) — Phase 2

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| 앱 래퍼 | **Tauri** (Electron 대체 — 10-15MB 번들, ~30MB RAM) |
| 3D 렌더링 | Three.js + glTF |
| 캐릭터 모델 | 미드저니 → Meshy AI 3D 변환 → glTF export |
| 애니메이션 | Mixamo 자동 리깅 또는 Blender 리깅 (fallback) |
| 충전 감지 | Rust `pmset -g batt` 파싱 (5초 폴링, 2초 debounce) |
| 창 설정 | Tauri 투명 프레임리스 + `set_ignore_cursor_events` (또는 동등 API) |

**왜 Tauri?** Electron은 기본 200MB+ 메모리, GPU 점유. 한손에 두고 쏘 행복한 룸메이트가 맥북을 느려지게 하면 본말이 전도. Tauri는 1/10 수준 리소스로 같은 결과.

## 로드맵

| 단계 | 작업 | 비고 |
|------|------|------|
| Step 1 | 미드저니로 희희 외모 확정 | 여러 앵글 레퍼런스 이미지 생성 |
| **Step 1.5** | **Meshy AI 모델 품질 검증** | **glTF export → Three.js 렌더 테스트. 품질 안 나오면 Rive 2D pivot** |
| Step 2 | Tauri + Three.js 앱 셋업 | 투명 창, alwaysOnTop, 클릭 패스스루 |
| Step 3 | 3D 모델 Three.js에 올리기 | 기본 AWAKE 상태 구현 |
| Step 4 | 충전 감지 연동 (Rust) | pmset 파싱 → CHARGING → NAPPING 전환 |
| Step 5 | 인터랙션 구현 | 쓰다듬기, 풀 먹이기 |
| Step 6 | 행동 풍부함 추가 | DROWSY (밤 11시) + 랜덤 하품 1종 |
| Step 7 | .dmg 패키징 + GitHub Releases 배포 | Tauri build |

### Step 1.5 품질 수용 기준 (3개 중 하나라도 실패 시 Rive 2D pivot)

1. Three.js 200×200px에서 아티팩트 없이 렌더링
2. 파일 크기 5MB 이하
3. Mixamo 또는 수동 리깅으로 idle + sleep 애니메이션 가능

## 에러 핸들링

- **GLTFLoader 실패:** fallback으로 정적 PNG 쿼카 이미지 표시 + 콘솔 에러 로그
- **충전 감지 실패:** `pmset` 파싱 실패 시 기본 AWAKE 상태 유지 (graceful degradation)
- **Tauri 투명 창 불안정:** fallback으로 CSS 투명 배경 + 최소 테두리 창
- **모델 로드 지연:** 로딩 중 정적 이미지 fade-in

## 창 관리 (macOS)

- **클릭 패스스루:** 투명 영역은 클릭 통과, 쿼카 렌더 영역만 클릭 캡처
- **풀스크린 앱:** alwaysOnTop 레벨을 fullscreen 아래로 설정
- **멀티 모니터:** Phase 1은 주 모니터 고정, 멀티 모니터 이동은 Phase 2
- **Mission Control / Spaces:** 모든 Space에 표시 (`NSWindow.collectionBehavior = .canJoinAllSpaces`, Tauri에서 `cocoa` crate 직접 호출 또는 plugin 활용)

## Success Criteria

- 맥북 코너에 희희가 떠있다
- 충전 꽂으면 풀밭 나오고 잠든다
- 충전 빼면 기지개 켜고 깨어난다
- 클릭하면 반응한다
- 런타임 메모리 100MB 이하, 번들 크기 30MB 이하
- 친구 한 명이 "귀엽다"고 말한다
