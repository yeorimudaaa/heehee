# 🌼 희희 (Heehee) — Happy Quokka Desktop Pet

> 맥북 코너에 조용히 살고 있는 픽사 스타일 3D 쿼카.
> 숫자도 없고, 닫기 버튼도 없고, 그냥 같이 있는 존재.

**타겟:** 본인 + 가까운 친구 몇 명 선물용. 공개 배포·상업화 목표 아님 — 제품이 아니라 선물.
**성공의 모양:** 내가 매일 켜둔다 + 친구가 "안 지웠다"고 말한다.

## 컨셉

희희는 게임 캐릭터가 아니라 **룸메이트**. 관리 대상이 아니라 같이 사는 존재.
내가 일하면 옆에서 나를 보고, 맥북 충전하면 희희도 쉰다.
수치도, 미션도, 알림도 없다. 그냥 있다.

## 상태 시스템

| 상태 | 트리거 | 행동 | 비고 |
|------|--------|------|------|
| `FIRST_GREETING` | **최초 실행 1회만** | 데이지를 화면 쪽으로 살짝 내밀며 인사 → 5초 후 AWAKE | 설정 플래그로 1회성 보장. 재설치해도 반복 X |
| `AWAKE` | 기본 / 충전 해제 후 / 앱 시작 시(비충전) | 화면 응시, 눈 깜빡임, 귀 쫑긋, 항상 웃음 | 앱 시작 시 충전 상태 확인 → 충전 중이면 CHARGING_IDLE로 |
| `HAPPY` | 쓰다듬기 / 풀 먹은 직후 | 통통 튀기, **데이지 꽃잎 이펙트** (3~5개 위로 떠오름), **데이지 살짝 내밀어 보이기** → 2초 후 AWAKE | 쓰다듬기 반응의 감정 포인트 |
| `IDLE_GESTURE` | AWAKE 상태에서 랜덤 (가끔) | 풀을 쳐다보며 "먹고 싶다" 제스처 → 안 줘도 AWAKE로 | 강제성 없음 |
| `YAWNING` | AWAKE 랜덤 (드물게) / DROWSY 중 자주 | 하품 1회 → AWAKE 복귀 | 체감 "살아있음" 강화 |
| `CHARGING_IDLE` | 충전 연결 직후 / 앱 시작 시(충전 중) | 풀밭 배경 fade-in, 털썩 앉아 멍때리기 | |
| `NAPPING` | 충전 5분 경과 | 눈 감고 낮잠, **쥐고 있던 아이템 꼭 쥔 채로**, ZZZ 이펙트 | |
| `STRETCHING` | 충전 해제 직후 | 기지개 + 눈 비비기 → AWAKE 복귀 | 양손 사용 → **꽃은 귀에 꽂혀 있음**. 복귀 시 손으로 |
| `DROWSY` | 밤 11시 이후 | 졸린 표정, YAWNING 빈도 증가 | **MVP 승격 (Step 6)** |
| `SURPRISED` | **이름 변경 시** | 깜짝 놀란 표정 → 2초 후 AWAKE | 개명 반응의 귀여움 |
| `SLEEPING_IN` | 트레이 "잠시 쉬게 하기" 선택 | 창 hide, 1시간 후 자동 복귀 | 유저 주권 존중 |

**Note:** 배고픔(HUNGRY) 게이지/타이머 없음. `IDLE_GESTURE`로 자연스럽게 표현.
**Debounce:** 충전 상태 변경은 2초 이내 재변경 무시 (MagSafe 빠른 탈착 대응).

### 쥐고 있는 아이템 (`held_item`)

희희는 **항상 꽃 한 송이를 가지고 있음**. 상태 시스템과 독립된 속성.

- 기본값: **데이지** 🌼
- **위치 규칙**:
  - 평소엔 **오른손**에 쥠
  - **양손을 쓰는 상태**(STRETCHING, 하품 중 손 드는 순간, SURPRISED 등)에선 **오른쪽 귀에 꽂음**
  - 상태가 끝나면 다시 손으로 복귀
- **풀 먹이기** → 데이지를 내려놓고 **풀을 쥐고 다님**
- **다음 날 아침** (로컬 자정 지나고 첫 AWAKE 진입 시) → **데이지로 복귀**
- NAPPING 중에도 가지고 있는 것 유지
- 저장 위치: 앱 로컬 상태 (`~/Library/Application Support/heehee/state.json`)

## 인터랙션

**유저가 할 수 있는 것**
- **쓰다듬기 (클릭)** → HAPPY + 데이지 살짝 내밀어 보이기
- **풀 먹이기 (버튼)** → HAPPY → 이후 풀을 쥐고 다님 (다음 날 아침 데이지 복귀). 안 줘도 슬퍼지지 않음
- **이름 짓기 / 바꾸기** (트레이 메뉴, 언제든) → 변경 시 SURPRISED 반응 한 번. 트레이 + 창 title에 표시
- **창 위치 이동 (드래그)** → 위치 기억. 다음 실행 시 마지막 위치 복원

### 트레이 메뉴 (감정 언어로)

닫기 버튼은 창엔 없지만, 메뉴바 트레이에 조용한 선택지를 둡니다.

- 🌼 **이름 짓기 / 바꾸기**
- 💤 **잠시 쉬게 하기** — 1시간 후 자동 복귀 (`SLEEPING_IN`)
- 🛏️ **이불 덮어주기** — 종료. 다음 실행까지 안녕
- (선택) 희희 정보 / 버전

> 철학 유지: "닫기"가 아니라 "돌봐주기". 유저 주권은 지키되, 룸메이트 톤으로.

**희희가 알아서 하는 것**
- **충전 감지** → CHARGING_IDLE → 5분 후 NAPPING
- **충전 해제** → STRETCHING → AWAKE
- **밤 11시 이후** → DROWSY (졸린 표정 + 하품 빈도 증가)
- **최초 실행** → FIRST_GREETING (1회만)
- **로컬 자정 경과** → 쥐고 있던 풀을 데이지로 교체

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| 앱 래퍼 | **Tauri** (Electron 대체 — 10-15MB 번들, ~30MB RAM) |
| 3D 렌더링 | Three.js + glTF |
| 캐릭터 모델 | 미드저니 → Meshy AI 3D 변환 → glTF export |
| 애니메이션 | Mixamo 자동 리깅 또는 Blender 리깅 (fallback) |
| 충전 감지 | Rust `pmset -g batt` 파싱 (5초 폴링, 2초 debounce) |
| 창 설정 | Tauri 투명 프레임리스 + `set_ignore_cursor_events` (또는 동등 API) |

**왜 Tauri?** Electron은 기본 200MB+ 메모리, GPU 점유. 한손에 두고 쏘 행복한 룸메이트가 맥북을 느려지게 하면 본말이 전도. Tauri는 1/10 수준 리소스로 같은 결과.

2. 파일 크기 5MB 이하
3. Mixamo 또는 수동 리깅으로 idle + sleep 애니메이션 가능

## 창 관리 (macOS)

- **클릭 패스스루:** 투명 영역은 클릭 통과, 쿼카 렌더 영역만 클릭 캡처
- **풀스크린 앱:** alwaysOnTop 레벨을 fullscreen 아래로 설정 — **의도**: 풀스크린 IDE·프레젠테이션 등 집중 순간엔 희희가 자동으로 물러남. 룸메이트 철학의 연장 (방해하지 않는 존재)
- **기본 위치:** 첫 실행 시 주 모니터 **우하단**. 드래그로 이동 시 위치 저장 → 다음 실행에서 복원
- **멀티 모니터:** Phase 1은 주 모니터 고정, 멀티 모니터 이동은 Phase 2
- **Mission Control / Spaces:** 모든 Space에 표시 (`NSWindow.collectionBehavior = .canJoinAllSpaces`, Tauri에서 `cocoa` crate 직접 호출 또는 plugin 활용)

## 📐 디자인 상세 사양

_2026-04-24 디자인 리뷰 반영 (D1~D10)_

**이펙트 & 애니메이션 파라미터**
- **D1. 풀밭 fade-in:** 2초 ease-out, 창 영역 한정, 최대 투명도 60%, 따뜻한 녹색 계열 (~#c8e4b5)
- **D2. HAPPY 이펙트:** **데이지 꽃잎** 3~5개, 2초 내 위로 떠올라 사라짐. 하트 이모지 ❌ — 희희만의 시그니처
- **D3. 빈도 (AWAKE 기준):**
  - `IDLE_GESTURE`: 약 20~30분당 1회
  - `YAWNING`: 약 45분당 1회 (DROWSY 중엔 15분당 1회)
- **D4. DROWSY 표현:** 눈 50% 감김, 몸 미세하게 앞뒤 흔들, YAWNING 15분당 1회

**에셋 사양**
- **D5. GLTFLoader 실패 시 fallback PNG:** Step 1.5에서 AWAKE 정면 포즈 1장을 참조 에셋으로 같이 export. 200×200 투명 배경 동일

**2D UI**
- **D6. 이름 다이얼로그 위치:** 희희 창 바로 위에 뜨는 **말풍선 모달**. 중앙 정렬 시스템 모달 X
- **D7. 이름 입력 검증:** 1~12자, 공백·개행 불가, 이모지 허용. 초과/빈 값이면 희희가 **머리를 갸우뚱** (에러 상태 시각화)

**접근성**
- **D8. 모션 감소 대응:** macOS 시스템 설정 "모션 줄이기" ON 시 — 통통 튀기·몸 흔들림 제거. 서사적 움직임(눈 깜빡임, 기지개, 하품)은 유지
- **D9. 다크 데스크탑 대응:** 눈동자 하이라이트가 어두운 배경에서 묻히지 않도록 캐릭터 외곽에 최소 1px 어두운 아웃라인

**친구 경험 설계**
- **D10. 친구 첫 만남 훅:** FIRST_GREETING은 본인만 봄 (친구 맥북엔 발생 X). 친구가 "귀엽다"를 뽑는 훅은 **쓰다듬기 반응** — 데이지 꽃잎 이펙트 + 데이지 살짝 내밀기 콤보

## Success Criteria

**기능적 목표**
- 맥북 코너에 희희가 떠있다
- 충전 꽂으면 풀밭 나오고 잠든다
- 충전 빼면 기지개 켜고 깨어난다
- 클릭하면 반응한다 (+ 데이지 살짝 보임)
- **밤 11시 이후엔 졸려 보인다**
- **이름 바꾸면 놀란다**
- **풀을 주면 풀을 쥐고 다니다가 다음 날 아침 데이지로 돌아간다**
- **트레이 메뉴에서 조용히 재울 수 있다**

**비기능적 목표**
- 런타임 메모리 100MB 이하, 번들 크기 30MB 이하
- 첫 실행 후 5초 안에 첫 렌더

**지속성 목표 (진짜 성공)**
- **내가 매일 켜둔다** (본인 지속성)
- **선물한 친구가 2주 후에도 안 지운다** (친구 지속성)
- 친구 한 명이 "귀엽다"고 말한다
