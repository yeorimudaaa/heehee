# 🌼 희희 (Heehee) — Happy Quokka Desktop Pet

> 맥북 코너에 조용히 살고 있는 픽사 스타일 3D 쿼카.  
> 숫자도 없고, 닫기 버튼도 없고, 그냥 같이 있는 존재.

---

## 캐릭터 시트

| 항목 | 내용 |
|------|------|
| 이름 | 희희 (Heehee) |
| 기본 표정 | 항상 웃음 — 입꼬리 올라간 픽사 스마일, 눈은 살짝 초승달 |
| 시그니처 아이템 | 민들레 꽃 한 송이 — 항상 오른손에. 상태가 바뀌어도, 낮잠 잘 때도 꼭 쥐고 있음 |
| 성격 | 긍정적이고 해맑음. 배고플 때도 꽃을 내밀며 애원하는 스타일 |
| 비주얼 | 픽사 스타일 3D — 큰 눈 하이라이트, 털 디테일, 통통한 비율 |

---

## 컨셉

희희는 게임 캐릭터가 아니라 **룸메이트**. 관리 대상이 아니라 같이 사는 존재.  
내가 일하면 옆에서 나를 보고, 맥북 충전하면 희희도 쉰다.  
수치도, 미션도, 알림도 없다. 그냥 있다.

---

## 상태 시스템

### 평소 (충전 미연결)

| 상태 | 트리거 | 행동 |
|------|--------|------|
| `AWAKE` | 기본 상태 | 화면 응시, 눈 깜빡임, 귀 쫑긋, 항상 웃음 |
| `HAPPY` | 쓰담쓰담 / 먹이 직후 | 통통 튀기, 하트 이펙트 |
| `HUNGRY` | 오래 먹이 안 줌 (내부 타이머) | 꽃 내밀며 애원, 배 붙잡기 |

### 충전 중 (휴식 모드)

| 상태 | 트리거 | 행동 |
|------|--------|------|
| `CHARGING_IDLE` | 충전 연결 직후 | 풀밭 배경 슬며시 fade-in, 털썩 앉아 멍때리기 |
| `NAPPING` | 충전 5분 경과 | 눈 감고 낮잠, 민들레 꼭 쥔 채로, ZZZ 이펙트 |
| `FRIEND_VISIT` | 충전 중 랜덤 이벤트 | 친구 쿼카 등장, 같이 뒹굴기 |
| `STRETCHING` | 충전 해제 직후 | 기지개 + 눈 비비기 → `AWAKE` 복귀 |

## 인터랙션

**유저가 할 수 있는 것**
- 쓰담쓰담 (클릭 또는 드래그)
- 풀 먹이기 (버튼 클릭)
- 이름 짓기 (최초 1회, 기본값: 희희)
- 창 위치 이동 (드래그)

**희희가 알아서 하는 것**
- 충전 감지 → 자동 휴식 모드 전환
- 충전 해제 → 기지개 후 AWAKE 복귀
- 시간대 반응 (밤 11시 이후 졸린 표정)
- 랜덤 이벤트 (친구 방문, 나비, 꽃 등)

## 비주얼

| 항목 | 내용 |
|------|------|

| 창 설정 | 항상 최상단 float, 닫기 버튼 없음 |

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| 앱 래퍼 | Electron |
| 3D 렌더링 | Three.js + glTF |
| 캐릭터 모델 | 미드저니 → Meshy AI 3D 변환 → glTF export |
| 애니메이션 | Blender 리깅 후 export (필요시) |
| 충전 감지 | Node.js `systeminformation` |
| 창 설정 | `alwaysOnTop` + 투명 프레임리스 윈도우 |

---

## 로드맵

| 단계 | 작업 | 비고 |
|------|------|------|
| Step 1 | 미드저니로 희희 외모 확정 | 여러 앵글 레퍼런스 이미지 생성 |
| Step 2 | Meshy AI로 3D 모델 변환 | glTF 파일 export |
| Step 3 | Electron + Three.js 앱 셋업 | 투명 창, alwaysOnTop 구현 |
| Step 4 | 3D 모델 Three.js에 올리기 | 기본 AWAKE 상태 구현 |
| Step 5 | 충전 감지 연동 | CHARGING → NAPPING 상태 전환 |
| Step 6 | 인터랙션 구현 | 쓰담쓰담, 먹이 주기 |
| Step 7 | 랜덤 이벤트 추가 | 친구 방문, 시간대 반응 등 |
